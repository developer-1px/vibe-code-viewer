# 경쟁사 및 유사 서비스 분석 보고서

## 조사 개요

본 보고서는 **Vibe Code Viewer**와 유사한 코드 의존성 시각화 및 분석 도구들을 조사하여 시장 현황, 기술 트렌드, 그리고 차별화 포인트를 파악하기 위해 작성되었습니다.

**조사 일자**: 2025년 1월
**조사 범위**: 코드 의존성 시각화, AST 기반 분석, 컴포넌트 관계 매핑 도구

## 1. 엔터프라이즈 상용 솔루션

### 1.1 CodeSee
- **웹사이트**: https://www.codesee.io
- **타겟**: 개발팀 전체 (모든 규모)
- **가격**: 무료 체험 6개월 제공, 구체적 가격 비공개

**주요 기능**:
- ✅ **Codebase Maps**: 크로스 레포지토리 및 서비스 의존성 시각화
- ✅ **Service Maps**: 서비스 간 데이터 흐름 및 아키텍처 이해
- ✅ **Visual Code Review Maps**: PR 머지 전 영향도 시각화
- ✅ **Interactive Code Tours**: 가이드 방식의 코드 탐색
- ✅ **Function Maps**: 함수 레벨 로직 및 의존성 분석
- ✅ **Code Automations**: 프로세스 및 베스트 프랙티스 자동 적용
- ✅ **CodeSee AI**: AI 기반 코드베이스 질의응답
- ✅ **Auto Documentation**: 코드 변경 시 실시간 문서 업데이트

**강점**:
- 엔터프라이즈 수준의 완성도
- AI 기반 자동화 및 문서화
- 팀 협업 중심 설계

**약점**:
- 가격 정보 불투명
- 대형 코드베이스 전용, 학습/교육 목적에는 과도할 수 있음
- 지원 언어/프레임워크 정보 부족

## 2. 오픈소스 CLI/라이브러리 도구

### 2.1 dependency-cruiser
- **GitHub**: https://github.com/sverweij/dependency-cruiser
- **NPM**: `dependency-cruiser`
- **타겟**: JavaScript/TypeScript 개발자
- **가격**: 무료 (오픈소스)

**주요 기능**:
- ✅ JavaScript, TypeScript, CoffeeScript 지원
- ✅ ES6, CommonJS, AMD 모듈 시스템 지원
- ✅ 의존성 검증 및 시각화
- ✅ DOT/SVG 그래프 출력
- ✅ 순환 의존성 탐지
- ✅ 커스텀 규칙 설정 가능

**사용 예시**:
```bash
npx depcruise src --include-only "^src" --output-type dot | dot -T svg > dependency-graph.svg
```

**강점**:
- 완전 무료 오픈소스
- 강력한 규칙 엔진
- 다양한 출력 포맷

**약점**:
- CLI 기반, 인터랙티브 UI 없음
- 정적 그래프만 생성 (실시간 편집 불가)
- Vue/React 컴포넌트 특화 기능 없음

- ✅ **크로스 레포지토리**: 모노레포 및 마이크로서비스 아키텍처 지원
- ✅ **실시간 업데이트**: 코드 변경 시 자동 문서 및 그래프 갱신
- ✅ **협업 기능**: PR 리뷰, 팀 공유, 코드 투어
- ✅ **보안 중심**: 취약점 의존성 탐지 및 경고

## 5. Vibe Code Viewer 차별화 분석

### 5.1 현재 강점

| 특징 | Vibe Code Viewer | 경쟁사 |
|------|------------------|--------|
| **Vue + React 동시 지원** | ✅ | ❌ (대부분 단일 프레임워크 또는 범용) |
| **브라우저 기반 인터랙티브 UI** | ✅ | ❌ (CLI 도구들) / 💰 (상용 도구들) |
| **실시간 코드 편집 + 시각화** | ✅ | ❌ (대부분 정적 그래프) |
| **무료 오픈소스** | ✅ | 일부 (CLI 도구들만) |
| **교육 목적 친화적** | ✅ | ❌ (대부분 엔터프라이즈 중심) |
| **AST 기반 정확한 분석** | ✅ | ✅ (일부 도구만) |
| **프레임워크 primitives 필터링** | ✅ | ❌ (찾지 못함) |
| **템플릿/JSX 스니펫 추출** | ✅ | ❌ (찾지 못함) |
| **커스텀 트리 레이아웃** | ✅ | ❌ (대부분 force-directed 또는 계층형) |

### 5.2 경쟁 우위

#### 🎯 포지셔닝: "교육 및 학습 중심 코드 시각화 도구"

**타겟 시장**:
1. **학습자**: Vue/React 입문자가 컴포넌트 의존성 이해
2. **교육자**: 강의/튜토리얼에서 코드 구조 설명
3. **온보딩**: 신규 팀원이 프로젝트 구조 빠르게 파악
4. **리팩토링**: 개발자가 의존성 영향도 분석

**차별화 포인트**:
- ✅ **즉시 사용 가능**: 설치 없이 브라우저에서 바로 사용
- ✅ **실시간 피드백**: 코드 편집 → 즉시 그래프 업데이트
- ✅ **프레임워크 특화**: Vue/React 컴포넌트 및 훅 전용 최적화
- ✅ **무료 & 오픈소스**: 학습/교육에 부담 없음
- ✅ **노이즈 제거**: Framework primitives 자동 필터링

### 5.3 현재 약점 및 개선 방향

**약점**:
- ❌ 로컬 프로젝트 업로드만 가능 (GitHub 통합 없음)
- ❌ 순환 의존성 탐지 미흡 (madge, dependency-cruiser에 비해)
- ❌ AI 기능 없음 (CodeSee에 비해)
- ❌ 팀 협업 기능 없음 (CodeSee, GitKraken에 비해)
- ❌ 다중 파일 프로젝트 지원 초기 단계

**개선 방향**:
1. **순환 의존성 탐지 및 하이라이트** (madge 벤치마킹)
2. **GitHub 레포지토리 직접 로드** (URL 입력 → 자동 분석)
3. **AI 기반 코드 설명** (Gemini API 활용 → 노드 클릭 시 설명)
4. **공유 기능** (그래프 URL 공유, 스크린샷 저장)
5. **성능 최적화** (대형 프로젝트 지원)
6. **다중 엔트리 포인트** (여러 컴포넌트 동시 분석)
